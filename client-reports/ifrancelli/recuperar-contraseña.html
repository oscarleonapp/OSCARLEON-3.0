Módulo recuperación de contraseña

Modal recuperar contraseña

  • Se creó la lógica de recuperación de contraseña utilizando una API REST.
  • Se creó la parte visual de la pantalla para poder realizar la recuperación de contraseña.
  • Se está trabajando en la lógica para enviar correos electrónicos de recuperación de contraseña desde el servidor SMTP de Industrias Francelli.
  • Se creó la página y componentes necesarios de la aplicación para manejar el proceso de recuperación de contraseña. Esto incluye una página para solicitar el restablecimiento de contraseña, una página para confirmar la solicitud y una página para ingresar la nueva contraseña.
  • Implementación de la lógica necesaria para generar tokens de restablecimiento de contraseña, enviar correos electrónicos con enlaces de restablecimiento y verificar y cambiar contraseñas.
  • Manejo adecuado de los errores que puedan surgir durante el proceso de recuperación de contraseña, como direcciones de correo electrónico no válidas y errores de envío de correo electrónico.
  • Se han realizado pruebas exhaustivas para asegurar que el proceso de recuperación de contraseña funcione como se espera en diferentes escenarios y condiciones.

Página de inicio

Se incluye en la pantalla de bienvida al sistema la opción de poder recuperar contraseña.

Modal Home Recuperar

Tareas a realizadas.

  • Crear el componente modal: Creación del componente modal que contendrá el formulario de recuperación de contraseña.
  • Diseñar el formulario: Formulario de recuperación de contraseña con los campos necesarios, como el campo de correo electrónico.

Validación de correo

Tareas a realizadas.

  • Agregar validación de datos: Validación de datos por .NET para validar el campo de correo electrónico en el formulario.
  • Manejar la lógica del formulario: Se define un método de manejo de envío del formulario (HandleSubmit) que se ejecutará cuando el usuario envíe el formulario. En este método, se implementa la lógica necesaria para procesar la solicitud de recuperación de contraseña, como enviar un correo electrónico al usuario con un enlace de recuperación.
  • Mostrar mensajes de validación: Se muestra un mensaje de error cuando el usuario ingresa un correo electrónico inválido o no ingrese ningún correo electrónico. Se manejará la validación automáticamente y se mostrará los mensajes de error como se puede apreciar en la imagen.
  • Abrir y cerrar el modal: Funcionalidad para abrir y cerrar el modal según sea necesario. El modal se abre cuando se hace clic en un botón recuperar contraseña y se cierra después de enviar el formulario de recuperación de contraseña con éxito.

Nueva contraseña

Tareas a realizadas.

  • Validar la nueva contraseña: Se utiliza .NET para validar el campo de la nueva contraseña en el formulario.

Validar repetición de contraseña

Tareas a realizadas.

  • Validar la repetición de la contraseña: Además de validar la nueva contraseña, también se necesita validar que la contraseña ingresada en el campo de repetir contraseña coincida exactamente con la nueva contraseña.
  • Manejar la lógica del formulario: Se define un método de manejo de envío del formulario que se ejecutará cuando el usuario envíe el formulario. En este método, además de enviar la solicitud de recuperación de contraseña, también se valida que la nueva contraseña y la repetición de la contraseña sean iguales. Si no coinciden, se muestra un mensaje de error apropiado al usuario.
  • Mostrar mensajes de validación: El formulario muestra mensajes de error adecuados cuando la nueva contraseña y la repetición de la contraseña no coincidan.
  • Actualizar la contraseña: Una vez que se haya validado la nueva contraseña y la repetición de la contraseña y ambas coincidan, se actualiza la contraseña del usuario en el sistema. Se hace una llamada a un método que actualiza la contraseña en la base de datos de autenticación.
